# 1. What is SDLC

**SDLC (System Development Life Cycle)** is a **structured process used
to design, develop, test, and maintain information systems or
software**.

Goal:

- develop **high-quality systems**
- reduce **development risk**
- ensure **user requirements are satisfied**

# 2. Phases of SDLC

    Planning
    ↓
    Requirement Analysis
    ↓
    System Design
    ↓
    Implementation
    ↓
    Testing
    ↓
    Deployment
    ↓
    Maintenance

# 3. Planning Phase

Purpose: **Identify the problem and determine project feasibility**

Activities:

- project planning
- defining scope
- resource estimation
- feasibility study

Types of feasibility:

  Type                      Meaning
  ------------------------- ----------------------
  Technical feasibility     technology available
  Economic feasibility      cost vs benefit
  Operational feasibility   organization ability

# 4. Requirement Analysis

Purpose: **Determine what the system must do**

Activities:

- gather user requirements
- interviews
- questionnaires
- observation

Outputs:

- requirement specification
- functional requirements
- non-functional requirements

Exam point:

    Requirement determination phase identifies what information is needed.

# 5. System Design

Purpose: **Decide how the system will work**

Design includes:

  Type              Description
  ----------------- ------------------------------
  Logical design    conceptual system model
  Physical design   actual system implementation

Tools used:

- Data Flow Diagrams (DFD)
- Entity Relationship Diagram (ERD)
- System architecture diagrams

# 6. Implementation (Coding)

Purpose: **Develop the actual system**

Activities:

- programming
- database creation
- module development

Examples:

- Java
- Python
- C++
- SQL

# 7. Testing Phase

Purpose: **Ensure the system works correctly**

Types of testing:

  Testing               Purpose
  --------------------- -------------------------
  Unit testing          test individual modules
  Integration testing   test module interaction
  System testing        test full system
  Acceptance testing    user validation

# 8. Deployment

Purpose: **Deliver system to users**

Types:

  Type                  Meaning
  --------------------- ----------------------------------
  Direct deployment     replace old system immediately
  Parallel deployment   run old and new systems together
  Phased deployment     implement step by step

# 9. Maintenance

Purpose: **Keep system running and improve it**

Types of maintenance:

  Type         Meaning
  ------------ -----------------------
  Corrective   fix errors
  Adaptive     adjust to environment
  Perfective   improve performance
  Preventive   prevent future issues

# 10. SDLC Models

These describe **how the phases are organized**.

    SDLC Models
    │
    ├─ Waterfall Model
    ├─ Spiral Model
    ├─ Iterative Model
    ├─ V-Model
    └─ Agile Model

# 11. Waterfall Model

Sequential development.

    Requirements
    ↓
    Design
    ↓
    Implementation
    ↓
    Testing
    ↓
    Maintenance

# 12. Agile Model

Iterative development.

Characteristics:

- rapid development
- continuous feedback
- flexible requirements

Used widely in **modern software development**.

# 13. Advantages of SDLC

- structured process
- better project control
- improved quality
- reduced risk

# 14. Disadvantages

- time consuming
- expensive
- less flexible in some models

500. The tools that support different stages of software development
life cycle are called: (a) CASE Tools (b) CAME tools (c) CAQE tools (d)
CARE tools

The correct answer is:

**(a) CASE Tools**

**Explanation:**\
**CASE Tools** stands for **Computer-Aided Software Engineering Tools**.
These tools help developers during different phases of the **Software
Development Life Cycle (SDLC)** such as planning, design, coding,
testing, and maintenance.

Other options are incorrect or not standard terms.
