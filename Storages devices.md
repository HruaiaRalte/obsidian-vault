# 1. BIT DEPTH (8-bit, 16-bit, etc.)

**Bit depth = number of bits used to represent a single unit of data.**

Used in:

- audio
- color
- processors
- digital signals

  Bit Depth   Possible Values
  ----------- ---------------------
  8-bit       256 values
  16-bit      65,536 values
  24-bit      16 million values
  32-bit      very high precision
  64-bit      modern CPUs

Example

    8-bit color
    = 256 colors

    = 16.7 million colors

356\. USB-type storage device is --

\(a\) Secondary (b) Auxiliary (c) Tertiary (d) Primary\
The correct answer is:

👉 **(b) Auxiliary**

### 💡 Explanation:

A **USB storage device (like a pen drive)** is considered **auxiliary
storage** because:

- It is **external** to the main computer system
- Used for **backup and transfer of data**

### 📌 Quick idea:

- **Primary storage** → RAM, ROM
- **Secondary storage** → Hard disk
- **Auxiliary storage** → External devices like USB drives ✅

✔️ Final answer: **Auxiliary**

Here's a **clear exam note** on **Auxiliary Storage** 👇

## 📦 Auxiliary Storage (External Storage)

### 💡 Definition

**Auxiliary storage** is a type of **secondary storage** that is
**external** to the computer and used to **store and transfer data**.

## 🔑 Key Features

- 📂 **Non-volatile** → data stays even when power is off
- 🔌 **Removable / portable**
- 💾 Used for **backup, sharing, and transfer**
- 🧳 Not always connected to the computer

## 🧠 Examples

- USB flash drive (pen drive)
- External hard disk
- Memory card (SD card)
- CD / DVD

## 🔄 Difference from other storage

  Type                    Example        Description
  ----------------------- -------------- ----------------------
  **Primary Storage**     RAM            Temporary, very fast
  **Secondary Storage**   Hard Disk      Permanent, internal
  **Auxiliary Storage**   USB, SD Card   External, portable

## ⚠️ Important Exam Point

👉 **Auxiliary storage = external secondary storage**\
👉 That's why USB devices are called **auxiliary storage** ✅

### 🧠 Explanation (simple)

In a magnetic disk (like a hard drive), data is arranged as:

- **Tracks** → circular rings on the disk ✅
- **Sectors** → small divisions of a track
- **Head** → reads/writes data
- **Block** → group of sectors

👉 **Circle = Track**\
👉 **Slice of circle = Sector**
