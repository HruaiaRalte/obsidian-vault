A **data bus** is the part of a computer’s system bus that is used to **transfer actual data** between components like the CPU, memory, and input/output devices.

---

### In simple terms:

It’s the **pathway that carries information** (numbers, instructions, results) inside the computer.

---

### 🔧 Key features:

- **Carries real data** (not instructions about what to do—that’s control bus)
    
- **Bidirectional** → data can move **both ways** (CPU ↔ Memory)
    
- **Width matters** → more bits = more data at once
    
    - 8-bit bus → 1 byte at a time
        
    - 32-bit bus → 4 bytes at a time
        
    - 64-bit bus → 8 bytes at a time
        

---

### 🧠 Easy analogy:

Think of the system like a delivery system:

- **Data bus** = trucks carrying goods 📦
    
- **Address bus** = destination address
    
- **Control bus** = traffic signals and instructions
    

---

### 📌 Example:

When you open a program:

- The **data bus** carries the program data from memory to the CPU
    
- The CPU processes it and may send results back via the data bus
